# 시스템 분류 규칙 (Homepage Classification)

포트폴리오 사이트에서 시스템 기획 항목을 분류할 때 사용하는 규칙입니다.
`content/projects/{id}/{폴더}/info.txt`의 `category:` 필드에 아래 분류 키워드를 사용합니다.

---

## 분류 목록

| category 값 | 한글명 | 설명 |
|---|---|---|
| Core | 코어 | 게임 전반에 공통 적용되는 처리 시스템 (보상, 효과, 효용, 자원 등) |
| Growth | 성장 | 캐릭터·장비·계정 능력치를 영구/일시적으로 증가시키는 시스템 |
| Trade | 교환 | 자원을 소비하여 다른 자원·아이템으로 변환하거나 교환하는 시스템 |
| Accomplishment | 달성 | 특정 행위/수준 도달 시 보상을 지급하는 동기부여 시스템 |
| Collect | 수집 | 다양한 요소를 획득·수집하는 과정에 재미를 부여하는 시스템 |
| Dungeon | 전투/던전 | PvE·PvP 전투 및 던전 관련 시스템 |
| Social | 소셜 | 플레이어 간 소통·관계·커뮤니티 관련 시스템 |
| Economy | 경제 | 재화·아이템의 거래 및 유통 관련 시스템 |
| Management | 경영 | 자원·공간·인력을 배치·관리하는 시뮬레이션 성격의 시스템 |
| Idle | 방치 | 오프라인 상태에서 자동으로 진행되는 시스템 |
| BM | BM | 현금/재화 기반 상품 구매 및 결제 관련 시스템 |
| Event | 이벤트 | 기간 한정 특별 행사 및 이벤트 관련 시스템 |
| UX | UX | 유저 편의성 향상을 위한 시스템 (즉시 완료, 자동 진행 등) |
| Guide | 가이드 | 유저 학습 지원 시스템 (튜토리얼, 획득처, 레드닷 등) |
| Service | 운영/서비스 | 계정·법률·SDK·운영툴 등 게임 외적 시스템 |
| World | 월드 | 게임 공간·환경·이동 관련 시스템 |
| Data | 데이터 | 시스템 기획 없이 데이터 입력·정리 작업만 수행한 항목 |
| Misc | 기타 | 위 분류에 속하지 않는 시스템 |

## 사용법

### info.txt 예시
```
name: 획득처
order: 2
category: Guide
roles: 시스템 기획, 데이터 입력
desc: 특정 자원에 대해 획득할 수 있는 콘텐츠를 안내하는 시스템입니다.
```

### 프로젝트 상세 페이지에서 category 기준으로 그룹핑하여 표시 가능

## etc 폴더 규칙

`content/projects/{id}/etc/` 폴더는 **사이트에 노출되지 않는 보관용** 폴더입니다.
- 이미지, 참고 자료, 작업물 등 첨부 가능
- build.py가 etc 폴더를 완전히 무시하므로 JSON에 포함되지 않음
- 개인 기록용 또는 미완성 작업 보관용으로 활용
